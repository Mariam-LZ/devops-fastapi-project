This repository provisions and configures a Kubernetes platform on Proxmox using Terraform and Ansible.

## рџљЂ Deployment workflow

### 1. Provision infrastructure with Terraform

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

### 2. Prepare Ansible inventory

Terraform writes the inventory to:

- `ansible/inventory/hosts.ini`
- `ansible/inventory/group_vars/all/generated.yml`

Do not edit `generated.yml` manually.

### 3. Configure secrets

Copy and edit the example vault file:

```bash
cp ansible/inventory/group_vars/all/vault.yml.example ansible/inventory/group_vars/all/vault.yml
```

Encrypt if needed:

```bash
ansible-vault encrypt ansible/inventory/group_vars/all/vault.yml
```

### 4. Run the Ansible deploy playbook

```bash
ansible-playbook -i ansible/inventory/hosts.ini ansible/playbooks/deploy.yml
```

## рџ“Њ Notes

- `terraform.tfstate` and `terraform.tfstate.backup` are generated files and ignore by `.gitignore`.
- `ansible/inventory/group_vars/all/generated.yml` is generated by Terraform.
- `ansible/inventory/group_vars/all/vault.yml.example` is the secret template.
- The install order in `ansible/playbooks/deploy.yml` ensures networking and certificates are deployed before storage and apps.
